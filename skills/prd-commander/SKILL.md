# /prd-commander

> 확정된 1Pager 기반으로 PRD → UI/UX 설계 → 와이어프레임 순서로 작성.
> Phase 1~3을 **자율 실행**한다 (승인 없이 자동 진행).
> **반드시 `/Users/amy/.claude/commands/kmong-context.md`를 읽고 공통 원칙을 적용한다.**
> 기존 CLAUDE.md 지침(한국어, 계획 먼저, 크몽 컨텍스트) 전체 상속.

---

## 실행 방법
```
/prd-commander [1Pager 파일명 또는 아이디어명]
/prd-commander --phase N       → 특정 Phase만 재실행 (이전 결과물 기반)
```

### 입력 파일 자동 감지
1Pager 파일이 지정되지 않으면:
1. `/Users/amy/Documents/문서/아이디어/*_1pager.md` 에서 가장 최근 수정된 파일을 탐색
2. 발견 시 → `📄 최근 1Pager 감지: [파일명] — 이 파일을 사용할까요?` 확인
3. 미발견 시 → `/idea-commander`를 먼저 실행하도록 안내

---

## 자율 실행 원칙

### 흐름
```
Phase 1 (PRD 작성) → Phase 2 (UI/UX 설계) → Phase 3 (와이어프레임) → 저장 + 다음 단계 안내
```

### 진행 상황 표시
```
⏳ Phase N — [Phase명] 진행 중...
✅ Phase N — [Phase명] 완료
```

**Phase 1~3은 승인 없이 자동 진행한다.** 사용자 확인 불필요.

## 1Pager → PRD 매핑 테이블

1Pager의 각 섹션이 PRD의 어떤 섹션으로 변환되는지 매핑. 누락 방지용.

| 1Pager 섹션 | PRD 매핑 대상 | 변환 방식 |
|-------------|-------------|----------|
| 1. 문제 정의 | 1. 개요 (배경 & 목적) | 문제를 배경으로 재구성 |
| 2. 솔루션 가설 | 1. 개요 (범위) + 4. 기능 요구사항 | 가설을 구체 기능으로 분해 |
| 3. 비즈니스 임팩트 | 2. 목표 & 성공 지표 | 지표를 측정 가능한 형태로 구체화 |
| 4. 검증 실험 설계 | 2. 목표 (목표 수치) + 8. 출시 전략 | 실험을 MVP/출시 계획으로 확장 |
| 5. 리스크 & 제약 | 5. 비기능 요구사항 + 6. 정책 & 예외 | 리스크를 기술/정책 제약으로 분리 |
| 6. RICE 스코어 | 7. 오픈 이슈 (우선순위 근거) | 우선순위 판단 근거로 활용 |
| 7. 전략적 포지셔닝 | 1. 개요 (경쟁 환경) | 차별점을 요구사항에 반영 |

PRD 작성 시 이 매핑을 체크하여 1Pager 내용이 빠짐없이 반영되었는지 확인한다.

## Phase 3 — 와이어프레임 (자동)

### 옵션 A: Figma MCP 연동 (우선 시도)

Figma MCP가 연결되어 있으면 `mcp__figma__generate_figma_design`으로 실제 와이어프레임을 생성한다.

- PRD의 기능 요구사항 + UI/UX 설계를 입력으로 전달
- 크몽 디자인 시스템 토큰 명시
- 생성 실패 시 옵션 B로 자동 전환

### 옵션 B: 텍스트 기반 와이어프레임 (폴백)

마크다운으로 화면 구조를 표현. 크몽 디자인 시스템 컴포넌트명 사용.

```markdown
## 와이어프레임

### [화면명]
**진입 조건:**
**레이아웃:** (컨테이너 1168px 기준)

---
[ Header ]
  로고 | GNB | CTA 버튼

[ 본문 영역 ]
  ┌─────────────────────────────┐
  │ [섹션 제목 — typo-24]       │
  │ [설명 텍스트 — typo-16]     │
  │                             │
  │  [ 컴포넌트명 ]              │
  │  [ 컴포넌트명 ]              │
  │                             │
  │  [ Primary Button — CTA ]  │
  └─────────────────────────────┘

[ Footer ]

**컴포넌트 스펙:**
- 컬러: kmong-context.md 토큰 참조
- 타이포: typo-24(제목), typo-16(본문), typo-14(캡션)
- 버튼: Primary(black) / Secondary(white)

**상태별 화면:**
- Default:
- Loading:
- Error:
- Empty:
```

## 저장

파일 저장 전 동일 아이디어명의 기존 파일 존재 여부를 확인한다:
- 기존 파일 있음 → `⚠️ 기존 파일 발견: [파일명]. 덮어쓸까요, 새 버전(_v2)으로 저장할까요?`
- 기존 파일 없음 → 정상 저장

- PRD 파일: `/Users/amy/Documents/문서/아이디어/YYYYMMDD_아이디어명_prd.md`
- 와이어프레임 파일: `/Users/amy/Documents/문서/아이디어/YYYYMMDD_아이디어명_wireframe.md`
