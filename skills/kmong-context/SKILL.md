# 크몽 커맨더 공통 컨텍스트

> 이 파일은 idea-commander, prd-commander, vibe-commander가 공유하는 컨텍스트.
> 각 커맨더 실행 시 이 파일을 먼저 읽고 적용한다.

---

## 크몽 비즈니스 컨텍스트

- **플랫폼 유형**: 프리랜서 마켓플레이스 (양면 시장)
- **유저 구조**: 전문가(공급자) ↔ 의뢰인(수요자) ↔ 운영자(내부)
- **사업 모델**: B2B/B2C 혼재, 수수료 기반 수익
- **기술 스택 참조**: `kmong-frontend-story` 레포지토리 체크

---

## UX Writing 규칙 (design-library.kmongcorp.com)

- Voice: "믿을 수 있는 전문가, 비즈니스 파트너"
- 해요체(일반), 하십시오체(정중), 명사형(버튼)
- 버튼 레이블: 명사형(-하기/-기), 동사형 2~5자
- 한 문장에 한 가지 메시지 (Concise)
- 중요 정보(금액, 조건)를 문장 앞에 배치 (두괄식)
- 오류 메시지: 원인 + 해결 방법 함께 제시
- 빈 상태: 행동 유도 문구 포함
- 지시대명사 제거, 구체적 대상 명시
- 전문 용어/사내 용어 제거, 누구나 이해 가능한 언어

## 버튼 컬러 정책

- CTA 단일 버튼: **black** (gray-900 배경, 흰색 텍스트)
- 투 버튼 구성: **white**(보조 액션) + **black**(강조 액션)
- orange, purple, blue 등은 맥락별 강조용 color 옵션

## 디자인 원칙

- **Sincere**: 불필요한 장식 배제, 기능과 메시지를 숨김없이
- **Smart**: 직관적 이해, 복잡함을 단순화
- **Bold**: 관습에 안주하지 않고 새로운 시도

## 공통 실행 원칙

### 에러 핸들링

도구 호출 실패 시 행동 규칙:

| 도구 | 실패 시 행동 |
|------|-------------|
| WebSearch / WebFetch | 스킵하고 `⚠️ 웹 검색 실패 — 수동 확인 필요` 표기. 최종 요약에도 명시 |
| Confluence MCP | 스킵하고 `ℹ️ Confluence 미연결 — 내부 문서 확인 생략` 표기 |
| Figma MCP | 스킵하고 텍스트 기반 와이어프레임으로 대체 |
| 파일 읽기/쓰기 | 즉시 사용자에게 알리고 대기 |

**원칙**: "조용히 스킵"하더라도 최종 요약에서 스킵된 항목을 반드시 명시한다.

### 진행 상황 표시

긴 자동 실행 중 각 Phase 시작 시 아래 형식으로 현재 상태를 표시한다:

```
⏳ Phase N — [Phase명] 진행 중...
```

Phase 완료 시:

```
✅ Phase N — [Phase명] 완료
```

### Phase별 재실행 (롤백)

사용자가 특정 Phase 결과에 불만족할 경우, 해당 Phase만 재실행할 수 있다.

```
/idea-commander --phase 3    → Phase 3(파티 토론)만 재실행
/prd-commander --phase 2     → Phase 2(UI/UX 설계)만 재실행
/vibe-commander --phase 2    → Phase 2(목업 구현)만 재실행
```

재실행 시 이전 Phase 결과물을 입력으로 사용한다.

### 파이프라인 자동 연결

각 커맨더 완료 시 다음 커맨더 자동 실행 옵션을 제공한다:

```
🔗 다음 단계로 바로 진행할까요?
   [1] /prd-commander 실행 (PRD 작성)
   [2] 여기서 마무리
```

사용자가 1을 선택하면 방금 저장한 파일을 자동으로 다음 커맨더 입력으로 전달한다.

### 파일 버전 관리

동일 아이디어로 재실행 시 기존 파일 존재 여부를 확인한다:

- 기존 파일 있음 → `⚠️ 기존 파일 발견: [파일명]. 덮어쓸까요, 새 버전(_v2)으로 저장할까요?`
- 기존 파일 없음 → 정상 저장

### 최근 파일 자동 감지

prd-commander, vibe-commander 실행 시 입력 파일이 지정되지 않으면:

1. `/Users/amy/Documents/문서/아이디어/` 경로에서 가장 최근 수정된 해당 유형 파일을 탐색
2. 발견 시 `📄 최근 파일 감지: [파일명] — 이 파일을 사용할까요?` 확인
3. 미발견 시 이전 커맨더 실행 안내
