## 테스트 2: 서비스카드 클렌징 1차 (PTBMAF)

### 배경 및 문제 정의

#### 우리의 고객
- **고객**: 서비스를 탐색하기 위해 크몽에 방문한 모든 구매 고객
- **JTBD**: 원하는 서비스를 쉽고 빠르게 탐색하여 적합한 전문가에게 빠르게 도달할 수 있어야 한다.

#### 단기 해결 문제 (1차 클렌징 목표)
- 온갖 라벨들이 썸네일에 위치해 복잡하게 느껴짐
- 중요 정보들이 아이콘 형태로 노출되어 빠르게 인지되지 않음
- 정보의 성격/우선순위에 따른 배치가 되어 있지 않음

**해결 방향**: 불필요 정보 제거, 아이콘 → 문장형 노출, 정보 우선순위에 따른 UI 재조정

#### 장기 해결 문제 (2차 이후)
- 카테고리 별 핵심 구매 요인이 서비스 카드에 전달되지 않음 (디자인: 포폴/클라, 마케팅: 리뷰, IT: 개발언어 등)
- 서비스 간 가격차에 대한 설득력 부족

---

### Tradeoffs

빠른 인지/비교가 가능해지면서 리스트 → 상세 전환율은 하락할 수 있으나, GMV 하락이 없는 한 긍정적 결과로 판단.

---

### Metric

> 1차 프로젝트는 '클렌징' 목적이 강하므로 정량적 지표 변화는 크지 않을 수 있음.
> 카테고리 별 커스텀 작업은 1차 클렌징 이후 진행.

1. 카테고리 리스트 → 서비스 클릭량 증가 (상세 전환율)
2. 카테고리 리스트 → 결제 전환율

## 테스트 3: 검색 결과 페이지 추천 테마 추가

### 배경 및 문제 정의

- **JTBD**: 검색 결과를 확인한 의뢰인은 서비스를 빠르게 탐색하여 원하는 결과에 도달해야 한다.
- **문제**: 연관도순 검색 결과만으로는 좋은 서비스를 변별하기 어렵다. (의뢰인은 인기 많은 서비스 or 평점 높은 서비스 순으로 결과를 확인하고 싶어함)

### Tradeoffs

- 검색 광고를 진행하는 전문가들의 반발 가능 (ex: "왜 얘들은 광고 없이 상위 노출?")

### Metric

| 지표 | 이벤트 |
|------|--------|
| 검색 결과 → 서비스카드 클릭 전환율 | `view_search_result (is_empty=F)` → `click_detailed_view (view_id=search_result)` |
| 검색 결과 → 문의하기 클릭 전환율 | `view_search_result (is_empty=F)` → `contactus_market_gig` |
| 검색 결과 → 구매 전환율 | `view_search_result (is_empty=F)` → `order_market_gig` |

## 테스트 22: 모바일 웹 헤더 개선

### 배경 및 문제

크몽 모바일 웹은 외부 채널(메타 광고, SEO, 인스타그램, 네이버 등)을 통해 신규 유저가 많이 인입된다. 신규 유저가 처음 만나는 화면은 상세 페이지 → 카테고리 순이나, 아래 세 가지 문제가 있다.

1. **크몽 플랫폼 인지 부족**: 바로 상세페이지로 진입하는 경우 크몽 아이덴티티가 보이지 않는다.
2. **높은 이탈률**: 모웹 상세 페이지 이탈률 70% 이상 (웹 50%). 뒤로가기 버튼 클릭 시 인입 채널(네이버, 광고 등)로 즉시 이탈한다.
3. **앱 설치 유도 부족**: 문의하기를 위해 앱 설치가 필요하나 유도 장치가 없다.

### 목적

1. 크몽 플랫폼 인지 (브랜딩/아이덴티티)
2. 모바일 웹에서 이탈 방지 & 탐색 이어가기 유도
3. 앱 설치 유도

### 요구사항

**웹투앱 배너**

| 항목 | 내용 |
|------|------|
| 노출 페이지 | 홈 / 카테고리 / 검색 / 상세 / 아티클(사업·마케팅 확인 필요) |
| 노출 제외 | 주문서 (주문 전환율 보호) |
| 기설치자 처리 | 앱 기설치자인 경우 앱 내 해당 페이지로 바로 이동 |

**모웹 상단 헤더 리디자인**
- 크몽 로고(kmong) 표기 & 뒤로가기 제거 → 플랫폼 인지 강화 & 홈/검색/카테고리 탐색으로 유도
- 브라우저 뒤로가기 또는 하단 닫기(X) 버튼을 통한 이전 화면 복귀는 허용

#### 3. 문의 = 구매의 핵심 게이트

문의 경험 여부에 따른 결제전환율 차이가 4.6배(26.7% vs 5.8%)이며, 신규 유저의 문의 클릭율이 4.23%로 매우 낮음. 이 갭을 줄이는 것이 비즈니스 임팩트가 가장 큰 영역.

- **유효**: 버튼 위치 개선(T15), 워딩 개선으로 부담감 해소(T26 ver.3)
- **비유효**: 클릭=즉시전송(T26 ver.1), 견적 Form 강제 노출(T30 ver.1~2)
- 핵심 원칙: "문의를 더 쉽게 시작하게" vs "문의에 더 많은 정보를 넣게"는 다름. 전자는 유효, 후자는 마찰만 증가.

#### 4. 전화운세: 마켓과의 cannibalization 관리가 핵심

- 전화운세 ↔ 마켓 운세 간 고객 이동이 실재함(T20, T25)
- 이탈 경로 차단(T20: 홈 아이콘 제거)이 충전전환 개선에 유효, 마켓 GMV 영향 없음
- 검색에서 전화운세 탭 추가(T25)도 마켓 GMV 영향 없이 전화운세 전환 개선
- 네이밍 변경(T27)은 진입 클릭까지만 유효, 실 상담 전환 미확인
- 테마 순서 변경(T29)은 효과 없음 → 전화운세는 UI 배치보다 "진입 경로 설계"가 더 중요한 도메인

#### 5. 원페이지 폼의 함정 (T31)

에어컨 청소 신청폼 원페이지 전환: 신청완료율 +12.1%(유의)이나 최종 결제전환은 동일. 마찰을 줄여 신청은 늘렸지만, 신중도가 낮은 유저까지 유입되어 결제로 이어지지 않음. → 퍼널 단순화가 항상 최종 매출로 연결되지는 않는다. "신청 완료"와 "결제 완료"는 다른 의사결정이며, 각 단계에 맞는 설득 요소가 필요.

#### 6. AI 활용은 큰 가능성 (T17, T28)

- AI 패키지 비교(T17): 패키지 공통점/차이점 요약 → 적용(Winner B)
- AI 의뢰서(T28): 등록전환 +11.1%, 유효인입률 66%(기존 47%), GMV +410%. 유효인입 AOV도 +194.5%. 단, 프롬프트 진입 후 등록 전환(20.6%)은 목표(30%) 미달 → AI가 만든 결과물의 정합성을 높이는 iteration이 필요.
